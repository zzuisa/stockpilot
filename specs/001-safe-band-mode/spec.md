# Feature Specification: 只赚不亏止损模式 + 波段选股适配性判断

**Feature Branch**: `001-safe-band-mode`
**Created**: 2026-07-06
**Status**: Draft
**Input**: 波段策略在模拟盘不稳定（如 NIO 5.02 买入后立刻 5.01 止损小亏）。算法不成熟期希望尽量少止损或不止损，保证每单只赚不亏（可少赚不可亏）。并新增选股能力：按指标判断标的是否适合做波段，不适合则提醒是否强制开启。

## Clarifications (2026-07-06 · /speckit-clarify)

- **灾难止损**：保留兜底——日常小浮亏不止损（只等止盈），但浮亏跌破 `disaster_stop_pct`（默认 **-25%**）仍市价止损，防退市/黑天鹅长期套牢。→ 解决 FR-009 与「灾难性下跌」边界。
- **生效范围**：safe_mode 开启后**即时对当前持仓也生效**（当前持仓立即停用亏损止损、只等止盈）。→ 解决「已运行策略即时 vs 仅新开仓」。
- **适配性严格度**：**适中**——综合波动/震荡度、趋势强度、流动性打分，明显单边趋势或流动性差判「不适合」，其余放行。→ 为 FR-006 阈值定基调（具体数值在 /speckit-tasks 落为默认可调参数）。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 只赚不亏止损模式（Priority: P1）

作为用手动/模拟盘验证策略的用户，我希望在策略不成熟阶段，开启一个「只赚不亏」模式：持仓**只在达到止盈目标时卖出**，不因小幅浮亏被立刻止损扫出（避免 5.02 买、5.01 卖的无谓亏损）。允许少赚、允许持仓等待，但单笔不产生亏损平仓。

**Why this priority**: 直接解决当前最痛的「买入即被止损、频繁小亏」问题，是稳定策略、建立信心的前提，单独交付即有价值。

**Independent Test**: 在模拟盘对一个标的开启该模式并制造一段小幅回撤行情，验证系统**不触发**任何亏损性卖出，只在价格回到止盈线时卖出获利。

**Acceptance Scenarios**:

1. **Given** 开启「只赚不亏」模式且持仓均价 5.02，**When** 价格回落到 5.01（浮亏 -0.2%）且指标偏空，**Then** 系统**不卖出**，继续持有等待止盈。
2. **Given** 开启该模式，**When** 价格涨到止盈目标（如均价 +0.5%），**Then** 系统按止盈卖出，记录为盈利单。
3. **Given** 该模式关闭（常规模式），**When** 浮亏触及止损线，**Then** 维持现有止损行为（向后兼容）。
4. **Given** 该模式开启，**When** 查看该策略状态，**Then** 能明确看到「只赚不亏」已启用及其风险提示。

---

### User Story 2 - 波段选股适配性判断（Priority: P2）

作为用户，在对某标的开启波段策略前，我希望系统**按指标判断它是否适合做波段**（例如足够的区间震荡/波动、不是单边强趋势、流动性足够），给出「适合 / 不适合」结论与理由。

**Why this priority**: 从源头减少「在不适合波段的标的上反复被套/被止损」的情况，是止损问题的上游治理。

**Independent Test**: 对一个明显单边下跌的标的和一个明显区间震荡的标的分别做适配性判断，验证前者判为「不适合」、后者判为「适合」，并给出可读理由。

**Acceptance Scenarios**:

1. **Given** 某标的近期为区间震荡、波动适中、流动性足，**When** 请求适配性判断，**Then** 返回「适合波段」+ 支撑指标理由。
2. **Given** 某标的处于强单边趋势或流动性极差，**When** 请求适配性判断，**Then** 返回「不适合波段」+ 理由。

---

### User Story 3 - 不适合时提醒并可强制开启（Priority: P2）

作为用户，当我要对一个被判为「不适合」的标的开启波段策略时，系统应**提醒风险并让我选择是否强制开启**，而不是直接拒绝或直接放行。

**Why this priority**: 保留用户最终决定权，同时把风险显性化；与 US2 共同构成完整的选股闸门。

**Independent Test**: 对「不适合」标的点击开启波段，出现二次确认浮窗；确认后策略以强制标记启动，取消则不启动。

**Acceptance Scenarios**:

1. **Given** 标的被判「不适合」，**When** 用户开启波段，**Then** 弹出「该标的不适合波段，是否强制开启？」确认；**And** 取消则不启动。
2. **Given** 用户在提醒中选择强制开启，**When** 启动，**Then** 策略启动且记录「强制开启（忽略适配性建议）」。

### Edge Cases

- 「只赚不亏」下若价格长期不达止盈 → 资金被长期占用：系统需如何呈现/提示「持仓滞留时长」？ [NEEDS CLARIFICATION: 是否需要「最长持有时长」软提醒，还是完全不干预]
- 真正的灾难性下跌（如 -30%、停牌/退市风险）是否仍完全不止损？ [NEEDS CLARIFICATION: 是否保留一个「灾难止损」硬上限作为兜底，还是严格零止损]
- 适配性数据不足（新标的、指标未补全）时，判为「不适合/未知」并按 US3 走强制确认。
- 已在运行的旧策略：开启「只赚不亏」是即时生效还是仅对新开仓生效。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 提供一个「只赚不亏」开关（每个策略独立），开启后**不因浮亏执行任何卖出**，仅在达到止盈目标时卖出。
- **FR-002**: 开启该模式时，系统 MUST 停用「指标止损」（浮亏+超卖+MACD 下行即卖）与「硬止损」两类亏损性平仓路径。
- **FR-003**: 系统 MUST 在策略状态与通知中明确标示「只赚不亏」已启用，并给出风险提示（可能长期持有/占用资金）。
- **FR-004**: 该模式 MUST 向后兼容——默认关闭，关闭时现有止损行为完全不变。
- **FR-005**: 系统 MUST 能对任一标的做「波段适配性判断」，输出结论（适合/不适合/未知）+ 可读理由。
- **FR-006**: 适配性判断 MUST 基于已采集的指标/行情（不新增外部数据源）。判据 SHOULD 包含波动/区间震荡程度、趋势强度、流动性，具体指标与阈值 [NEEDS CLARIFICATION: 采用哪些指标与阈值，如 ATR%、近 N 日振幅、趋势 regime、量比下限]。
- **FR-007**: 开启波段策略时，若标的被判「不适合」，系统 MUST 提醒并要求二次确认；用户可**强制开启**。
- **FR-008**: 系统 MUST 记录该策略是否为「强制开启（忽略适配性建议）」。
- **FR-009**: 「灾难止损」兜底是否保留 [NEEDS CLARIFICATION: 是否设一个极端跌幅硬止损上限（如 -25%）以防退市/黑天鹅，还是严格零止损]。

### Key Entities *(include if feature involves data)*

- **策略配置**：新增「只赚不亏」开关、「强制开启」标记、（可选）灾难止损上限。
- **适配性评估结果**：标的、结论（适合/不适合/未知）、评分与理由、评估时间。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 开启「只赚不亏」后，回放一段含小幅回撤的行情，**亏损性卖出次数 = 0**。
- **SC-002**: 该模式下所有平仓单的已实现盈亏 **≥ 0**（可少赚，不为负）。
- **SC-003**: 对已知「强单边趋势」与「区间震荡」样本各若干，适配性判断方向正确率 ≥ 80%。
- **SC-004**: 对「不适合」标的开启波段时，100% 出现二次确认，不会静默启动。
- **SC-005**: 默认关闭时，既有策略行为与开启本功能前**完全一致**（无回归）。

## Assumptions

- 面向模拟盘/策略验证阶段；「只赚不亏」是过渡期稳态工具，用户理解其代价（可能长期持有、资金占用、极端行情套牢）。
- 复用现有指标/行情与量化引擎，不引入新外部数据源。
- 适配性判断为**建议**性质，最终决策权在用户（可强制开启）。
- 默认关闭，向后兼容优先。
